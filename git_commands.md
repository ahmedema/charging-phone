# أوامر الجيت هب الأساسية لرفع التعديلات لحقاً

بمجرد إصلاح مجلد الجيت الأساسي المخفي `.git` في مشروعك، يمكنك اتباع الخطوات التالية في نافذة الأوامر Terminal لرفع التحديثات مستقبلاً.

## 1. تحديد الملفات التي تم تعديلها
لإضافة التعديلات في جميع الملفات دفعة واحدة:
```powershell
git add .
```
أما إذا أردت رفع التعديلات في **ملف واحد فقط أو ملفات محددة** (مثلاً ملف .env أو ملف src/main.js):
```powershell
git add اسم_الملف_أو_المسار
# مثال:
# git add src/main.js
```

## 2. حفظ التعديلات وإعطائها وصفاً (Commit)
هذا الأمر يحفظ التعديلات محلياً ويكتب رسالة تشرح ما الذي تم تعديله:
```powershell
git commit -m "اكتب هنا وصفاً مختصرًا لتحديثاتك"
# مثال:
# git commit -m "تعديل واجهة المستخدم وإضافة ألوان جديدة"
```

## 3. رفع التعديلات إلى الجيت هب (Push)
لرفع التعديلات التي تم حفظها إلى المستودع الخاص بك على الإنترنت:
```powershell
git push
```

---

## 💡 نصيحة هامة بخصوص مشكلتك الحالية:
بسبب مشكلة مجلد `.git` القديم والتالف في الوقت الحالي، إن أردت رفع أي شيء **قبل** أن تفرغ من إصلاح هذا المجلد وعمل Git عادي، فسيتوجب عليك استخدام الأوامر بصيغتها المؤقتة التي استخدمناها:

```powershell
git --git-dir=F:\temp_git_for_charging_phone\.git --work-tree=F:\charging_phone add .
git --git-dir=F:\temp_git_for_charging_phone\.git --work-tree=F:\charging_phone commit -m "وصف التحديث"
git --git-dir=F:\temp_git_for_charging_phone\.git --work-tree=F:\charging_phone push
```
ولكن بمجرد نقلك لمحتوى المجلد المؤقت ليصبح مجلد `.git` العادي لمشروعك، ستتمكن من استخدام الأوامر الثلاثة البسيطة المتواجدة في أعلى هذا الملف.
